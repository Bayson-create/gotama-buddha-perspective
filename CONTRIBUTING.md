# 贡献指南 / Contributing Guide

感谢你对佛陀Skill项目的兴趣！欢迎各种形式的贡献。

Thank you for your interest in the Gotama Buddha Perspective Skill! We welcome contributions of all kinds.

## 🤝 贡献方式 / How to Contribute

### 报告问题 / Report Issues
- 发现bug或有建议？[提交Issue](https://github.com/huashu/gotama-buddha-perspective/issues)
- Found a bug or have a suggestion? [Submit an Issue](https://github.com/huashu/gotama-buddha-perspective/issues)
- 请详细说明：问题描述、复现步骤、期望结果
- Please include: description, steps to reproduce, expected outcome

### 改进框架 / Improve the Framework
- 提交PR前，请先在Issue中讨论你的想法
- Before submitting a PR, please discuss your ideas in an Issue first
- 确保改动与巴利三藏的学术准确性一致
- Ensure changes maintain academic accuracy with the Pali Canon

### 补充研究资料 / Add Research Materials
- 发现新的学术资料或翻译？欢迎补充 `references/research/` 中的文件
- Found new academic materials or translations? Welcome to supplement files in `references/research/`
- 标注清晰的来源和可信度 / Mark sources and credibility clearly

### 翻译 / Translation
- 欢迎翻译为其他语言的版本
- Welcome to translate into other languages
- 创建新的 `SKILL.[lang].md` 文件（如 SKILL.en.md）
- Create a new `SKILL.[lang].md` file (e.g. SKILL.en.md)

## 📝 PR提交流程 / Pull Request Process

1. **Fork** 此仓库
2. **创建分支**：`git checkout -b feature/your-feature-name`
3. **提交改动**：`git commit -m "简要描述你的改动"`
4. **推送分支**：`git push origin feature/your-feature-name`
5. **开启PR**：在GitHub上创建Pull Request，说明改动内容和原因

## 📋 PR检查清单 / PR Checklist

在提交PR前，请确认：

- [ ] 改动与巴利三藏或已验证的学术资料对应
- [ ] 没有破坏现有的心智模型或决策启发式的逻辑一致性
- [ ] 新增的诚实边界真实反映了局限
- [ ] 如果修改了SKILL.md，确保表达DNA特征仍然一致
- [ ] 更新了相关的reference文件
- [ ] 文档清晰易懂

## 🎓 质量标准 / Quality Standards

### 心智模型 / Mental Models
- 必须有≥2个不同领域的历史证据
- Must have evidence from ≥2 different domains from history
- 必须标注失效条件 / Must state when it fails

### 决策启发式 / Decision Heuristics  
- 必须有具体的历史案例支撑
- Must be supported by specific historical examples
- 必须说明适用场景 / Must clarify applicable scenarios

### 表达DNA / Expression DNA
- 维持10个特征的一致性 / Maintain consistency of 10 features
- 不破坏"能在100字内认出是佛陀"的辨识度
- Don't break the "recognizable as Buddha in 100 chars" quality

### 诚实边界 / Honest Boundaries
- 每条局限必须真实且具体 / Each must be real and specific
- 不允许掩盖矛盾，应记录为"已知张力"
- Don't hide contradictions, record as "known tensions"

## 🙏 社区规范 / Community Guidelines

- 尊重所有参与者
- 关注学术准确性，超越政治或宗教立场
- Focus on academic accuracy beyond political or religious positions
- 欢迎不同观点，但要基于证据讨论
- Welcome different perspectives, but discuss based on evidence

## ❓常见问题 / FAQ

**Q: 我能添加新的心智模型吗？**
A: 可以，但需要：
1. 从巴利经文中找到≥2个明确的证据
2. 在Issue中讨论和验证
3. 与现有6个模型的逻辑一致

**Q: 能改变SKILL.md的结构吗？**
A: 结构改变需要在Issue中充分讨论，因为会影响角色扮演规则的执行。

**Q: 如何处理历史记载中的矛盾？**
A: 记录为"已知张力"，说明不同的学者或传承如何理解，而不是隐瞒。
